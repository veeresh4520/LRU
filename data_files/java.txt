java=Object oriented programming language designed for portability and robustness
jvm=Java virtual machine executing compiled bytecode
jre=Java runtime environment providing libraries and jvm
jdk=Java development kit containing compiler and development tools
bytecode=Intermediate code generated by java compiler
compiler=Program translating source code into bytecode
class=Blueprint defining properties and behavior of objects
object=Instance of a class with state and behavior
method=Function defined inside a class
constructor=Special method used to initialize objects
main=Entry point method of a java application
package=Namespace organizing related classes and interfaces
import=Keyword allowing access to classes from other packages
variable=Named memory location storing data
data type=Specification of type of data a variable can hold
primitive=Basic built in data type storing simple values
reference=Variable storing address of an object
int=Integer primitive data type
double=Floating point primitive data type
char=Primitive data type representing single character
boolean=Primitive data type storing true or false
string=Class representing sequence of characters
array=Fixed size collection of elements of same type
operator=Symbol performing operation on operands
expression=Combination of variables and operators producing value
statement=Complete instruction executed by program
if=Conditional statement executing code based on condition
switch=Selection statement choosing execution path
loop=Control structure repeating code block
for=Loop executing fixed number of iterations
while=Loop executing while condition remains true
do while=Loop executing at least once before condition check
break=Statement terminating loop or switch
continue=Statement skipping to next loop iteration
return=Statement sending value back from method
object oriented=Programming paradigm based on objects and classes
encapsulation=Wrapping data and methods into single unit
abstraction=Hiding implementation details from user
inheritance=Mechanism allowing class to acquire properties of another
polymorphism=Ability of methods to take multiple forms
overloading=Defining multiple methods with same name different parameters
overriding=Providing new implementation of inherited method
super=Keyword referring to parent class
this=Keyword referring to current object
interface=Contract defining abstract methods to implement
abstract class=Class that cannot be instantiated and may contain abstract methods
final=Keyword preventing modification or inheritance
static=Keyword belonging to class rather than instance
access modifier=Keyword controlling visibility of classes and members
public=Access level allowing visibility everywhere
private=Access level restricting visibility to same class
protected=Access level allowing subclass access
exception=Event disrupting normal program flow
try=Block enclosing code that may throw exception
catch=Block handling thrown exception
finally=Block executing regardless of exception
throw=Statement explicitly throwing exception
throws=Keyword declaring possible exceptions
checked exception=Exception checked at compile time
unchecked exception=Exception occurring at runtime
multithreading=Ability to execute multiple threads concurrently
thread=Lightweight unit of execution
runnable=Interface representing task executed by thread
synchronization=Mechanism controlling concurrent access
collection=Framework providing data structures like lists and sets
list=Ordered collection allowing duplicates
set=Collection disallowing duplicate elements
map=Collection storing key value pairs
arraylist=Resizable array implementation of list
linkedlist=List implementation using linked nodes
hashmap=Map implementation using hashing
iterator=Object used to traverse collection elements
generics=Feature enabling type safety in collections
lambda=Concise function expression introduced in java eight
stream=Api supporting functional style operations on collections
file=Representation of file path in java
io=Package handling input and output operations
serialization=Process converting object into byte stream
deserialization=Process reconstructing object from byte stream
jdbc=Api enabling database connectivity in java
