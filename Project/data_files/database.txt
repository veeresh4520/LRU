database=Organized collection of structured data stored electronically
data=Raw facts and figures without context
information=Processed data that has meaning and usefulness
dbms=Software system used to create manage and manipulate databases
rdbms=Database management system based on the relational model
schema=Logical structure defining tables relationships and constraints
instance=Actual data stored in the database at a specific time
metadata=Data that describes the structure and properties of other data
catalog=System table storing metadata about database objects
data model=Conceptual framework describing how data is organized
relational model=Data model representing data in tables with relations
table=Structured collection of rows and columns
relation=Mathematical term for a table in relational databases
tuple=Single row representing a record in a table
attribute=Column representing a property of an entity
domain=Set of valid values for an attribute
primary key=Attribute that uniquely identifies each record in a table
candidate key=Minimal attribute set that can uniquely identify records
superkey=Attribute set that uniquely identifies records possibly with extra fields
composite key=Primary key formed using multiple attributes
foreign key=Attribute linking one table to another table primary key
referential integrity=Constraint ensuring foreign keys correctly reference primary keys
entity=Real world object represented in a database
entity set=Collection of similar entities
relationship=Association between two or more entities
relationship set=Collection of similar relationships
er diagram=Graphical representation of entities and relationships
weak entity=Entity that depends on another entity for identification
participation=Degree to which an entity is involved in a relationship
normalization=Process of organizing data to reduce redundancy
normal form=Level of database normalization based on rules
first normal form=Form ensuring atomic values and no repeating groups
second normal form=Form removing partial dependency on composite keys
third normal form=Form removing transitive dependency
bcnf=Stronger normal form ensuring every determinant is a candidate key
functional dependency=Relationship where one attribute determines another
partial dependency=Dependency on part of a composite key
transitive dependency=Indirect dependency through another attribute
decomposition=Breaking a table into smaller tables to reduce redundancy
redundancy=Unnecessary duplication of data
anomaly=Problem caused by redundancy during insert update or delete
sql=Standard language used to interact with relational databases
ddl=Commands used to define database structures
dml=Commands used to manipulate data within tables
dcl=Commands used to control user access and permissions
tcl=Commands used to manage transactions
select=Command used to retrieve data from tables
insert=Command used to add new records to a table
update=Command used to modify existing records
delete=Command used to remove records from a table
where=Clause used to filter records based on conditions
join=Operation combining rows from multiple tables
inner join=Join returning matching records from both tables
left join=Join returning all records from left table and matches from right
right join=Join returning all records from right table and matches from left
full join=Join returning all records when there is a match in either table
group by=Clause used to group rows with similar values
having=Clause used to filter grouped records
order by=Clause used to sort query results
subquery=Query nested inside another query
view=Virtual table based on the result of a query
index=Data structure that improves data retrieval speed
constraint=Rule enforcing data integrity in a table
unique=Constraint ensuring all values in a column are distinct
not null=Constraint preventing null values in a column
check=Constraint enforcing specific conditions on data
default=Constraint assigning a default value to a column
transaction=Sequence of operations treated as a single logical unit
acid=Set of properties ensuring reliable transaction processing
atomicity=Property ensuring a transaction completes fully or not at all
consistency=Property ensuring database remains valid after transaction
isolation=Property ensuring transactions do not interfere improperly
durability=Property ensuring committed changes persist permanently
concurrency=Ability of database to handle multiple transactions simultaneously
locking=Mechanism controlling concurrent access to data
shared lock=Lock allowing multiple transactions to read data
exclusive lock=Lock allowing only one transaction to modify data
deadlock=Situation where transactions wait indefinitely for each other
timestamp=Value used to order transactions in concurrency control
recovery=Process of restoring database after failure
log=Record of database changes used for recovery
checkpoint=Point where database state is saved for faster recovery
rollback=Operation that undoes uncommitted transaction changes
commit=Operation that permanently saves transaction changes
storage engine=Component responsible for storing retrieving and managing data
buffer=Memory area used to cache database pages
page=Fixed size block of data stored on disk
record=Collection of related fields representing one entity instance
indexing=Technique of creating indexes to speed up queries
btree=Balanced tree structure commonly used for indexing
hashing=Technique using hash functions for fast data retrieval
partitioning=Dividing a table into smaller segments for performance
sharding=Distributing data across multiple servers
replication=Copying data across multiple systems for availability
backup=Process of creating copies of database data
restore=Process of recovering data from backups
nosql=Non relational database model designed for scalability
document=Nosql data model storing data as structured documents
key value=Nosql model storing data as key and corresponding value pairs
column family=Nosql model storing data in column oriented groups
