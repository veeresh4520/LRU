operating system=System software managing hardware resources and providing services to applications
kernel=Core component of operating system controlling hardware access
shell=Interface allowing users to interact with operating system
process=Program in execution with its own memory space
program=Set of instructions stored on disk
thread=Smallest unit of execution within a process
multitasking=Ability to run multiple processes concurrently
multiprocessing=Use of multiple processors to execute tasks
context switch=Saving and restoring process state during switching
scheduler=Component deciding which process runs next
scheduling=Method of allocating cpu time to processes
fcfs=Scheduling algorithm serving processes in arrival order
sjf=Scheduling algorithm selecting shortest job first
round robin=Scheduling algorithm giving each process equal time slice
priority scheduling=Scheduling based on process priority levels
cpu=Central processing unit executing instructions
interrupt=Signal requesting immediate attention from cpu
system call=Interface for programs to request os services
mode switch=Transition between user mode and kernel mode
user mode=Restricted mode for application execution
kernel mode=Privileged mode for operating system execution
memory=Primary storage used by cpu for active processes
ram=Volatile memory storing temporary data
virtual memory=Technique using disk space to extend ram
paging=Memory management dividing memory into fixed size pages
page=Fixed size block of virtual memory
frame=Fixed size block of physical memory
page table=Data structure mapping pages to frames
segmentation=Memory management dividing memory into logical segments
fragmentation=Wasted memory space due to allocation strategy
thrashing=Excessive paging causing performance degradation
deadlock=Situation where processes wait indefinitely for resources
starvation=Indefinite postponement of process execution
critical section=Code segment accessing shared resources
mutex=Lock ensuring mutual exclusion in critical section
semaphore=Synchronization primitive controlling resource access
monitor=High level synchronization construct
ipc=Mechanism enabling interprocess communication
shared memory=Ipc method using common memory space
message passing=Ipc method exchanging messages between processes
pipe=Ipc mechanism connecting output of one process to input of another
socket=Endpoint for network communication between processes
file system=Method of organizing and storing files on storage devices
file=Collection of related data stored on disk
directory=Structure organizing files in hierarchical manner
inode=Data structure storing file metadata
permission=Access rights controlling file usage
buffer=Temporary memory area for data transfer
cache=High speed storage improving data access time
disk=Secondary storage device for persistent data
block=Smallest unit of disk storage allocation
booting=Process of starting operating system
bootloader=Program loading operating system into memory
device driver=Software enabling communication with hardware devices
virtualization=Creating virtual versions of hardware resources
hypervisor=Software managing virtual machines
process state=Current status of a process
ready state=Process waiting for cpu allocation
running state=Process currently executing
waiting state=Process waiting for event or resource
zombie process=Terminated process awaiting cleanup
orphan process=Process whose parent has terminated
priority=Level determining scheduling preference
throughput=Number of processes completed per unit time
turnaround time=Total time from submission to completion
waiting time=Time process spends waiting in ready queue
response time=Time from request submission to first response
load balancing=Distributing workload evenly across processors
concurrency=Managing multiple processes during overlapping time
parallelism=Executing multiple tasks simultaneously
race condition=Unexpected behavior due to unsynchronized access
spinlock=Lock using busy waiting technique
context=Stored information representing process state
memory allocation=Assigning memory space to processes
heap=Memory region for dynamic allocation
stack=Memory region for function calls and local variables
swapping=Moving processes between memory and disk
protection=Mechanism preventing unauthorized resource access
security=Measures protecting system from threats
authentication=Verifying identity of user or process
authorization=Granting access permissions to resources
